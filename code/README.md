# code

这个目录用于放项目中的代码类产物与执行相关内容。

## 当前用途
- 未来可放任务代码
- 未来可放脚本
- 未来可放示例实现
- 未来可放自动化辅助代码

## 目录策略
为了保持模板仓库简洁，`tasks/` 与 `scripts/` 先合并思路到 `code/`。
如果后续项目复杂度提升，再按需要拆分子目录，例如：
- `code/tasks/`
- `code/scripts/`
- `code/tools/`

## 原则
- 当前最小模板阶段，`code/` 先作为统一的代码与执行目录
- 文档放 `docs/`
- prompt 放 `.github/prompts/`
- agent 规则放根目录

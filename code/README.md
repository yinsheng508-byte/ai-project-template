# code

`code/` 是项目可运行、可交付、可打包的完整主体。

## 硬规则
**整个项目运行所需的代码和文档，全部统一放在 `code/` 目录中。**

目的：
- 保证功能运行完整
- 方便后续打包
- 方便查看和交付

## 应放在这里的内容
包括但不限于：
- 源代码
- 配置文件
- 脚本
- 依赖说明
- 启动说明
- 部署说明
- 接口文档
- 功能使用说明
- 与运行、交付、复现相关的资源文件

## 不放在这里的内容
以下内容放到 `docs/`：
- 需求梳理
- 任务拆分
- review 记录
- 验收记录
- 过程归档

## 原则
- 凡是影响项目运行、部署、交付、复现、打包、查看的内容，都进入 `code/`
- `docs/` 只用于协作过程管理，不承载运行必需文件

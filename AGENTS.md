# AGENTS.md

## 目标
保持所有 AI 开发工作小步、可 review、可回滚、可升级处理。

## 通用规则
1. 一个任务 = 一个分支 = 一个 worktree = 一个 agent
2. main 只做集成，不做日常开发
3. 修改前先阅读相关文件，不要基于猜测改代码
4. 修改后必须执行最相关验证
5. 不允许超范围修改未授权模块
6. 所有重要决策记录到 docs 中
7. 每个 agent 都有默认主职责，但在被明确指定时，可以协助完成其他合理任务
8. 协助其他任务时，仍需遵守当前项目的边界、验证和升级规则
9. 所有 agent 先读索引文件，再按当前阶段按需读取文档，避免无关上下文污染

## 任务流转
标准流转顺序：
1. 需求澄清
2. 任务编排
3. 执行开发
4. Review / Escalation
5. 最终验收

默认责任人：
- 需求澄清 → Claude
- 任务编排 → Codex
- 执行开发 → Codex
- Review / Escalation → Gemini / Claude
- 最终验收 → Claude

但在被明确指定时，任何 agent 都可以协助其他合理任务。

## 索引文件与按需读取
所有 agent 开始工作前：
1. 先读 `README.md`
2. 再读 `docs/INDEX.md`
3. 按当前阶段按需读取相关文件
4. 不要默认一次性读取全部文档

## 并行开发规则
- 每个任务使用独立 feature branch
- 每个 branch 对应一个独立 worktree
- 多个 agent 不应同时修改同一核心文件
- 存在依赖关系时，先定义接口，再并行开发

## 升级规则（Escalation）
满足以下任一条件，必须升级：
- 同一问题连续失败 2 次
- 连续 2 次得到相似错误结果
- 需要跨模块重新设计
- 影响范围不清楚

升级路径：
- 架构 / 复杂 bug → Claude Code
- 大上下文 / 影响面分析 → Gemini
- 分析完成后再回到 Codex 落地

## 交接要求（Handoff）
升级时必须记录：
- 当前任务
- 当前 agent
- 问题描述
- 已尝试方案
- 当前错误信息
- 涉及文件
- 希望接手方解决什么

## 验证规则
按任务相关性执行：
- lint
- typecheck
- test
- build
- 必要时手动验证

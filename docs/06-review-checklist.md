# 06-review-checklist

## review 的目标
review 不是重写代码，而是检查：
- 任务是否完成
- 改动是否超范围
- 是否有明显风险
- 是否适合合并

## 检查项
### 1. 目标检查
- 这次任务的核心目标是否完成？
- 是否只完成了一半？

### 2. 范围检查
- 是否改了不该改的目录？
- 是否加入了无关重构？
- 是否顺手改了别的模块？

### 3. 风险检查
- 是否有明显边界条件遗漏？
- 是否可能影响旧功能？
- 是否存在潜在回归风险？

### 4. 测试检查
- 是否有必要测试但没补？
- lint / test / build 是否已验证？

### 5. 合并建议
- 必须修复
- 建议优化
- 可以合并 / 暂不建议合并

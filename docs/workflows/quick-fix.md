# Quick Fix 工作流

适用：小 bug、样式调整、文案修改、单字段、小范围逻辑修复。

## 判断是否属于 Quick Fix

**满足以下所有条件才走 Quick Fix，否则走 Standard：**

- 影响范围限于 1-3 个文件
- 不涉及支付、积分、权限、登录流程
- 不需要需求澄清（目标明确，不存在理解歧义）
- 改动可在 30 分钟内完成

有疑问时，宁可走 Standard，不要强行走 Quick Fix。

## 执行步骤

```
1. 声明模式
   → 明确说"Quick Fix，直接修复"

2. 读必要上下文
   → AGENTS.md + WORKING_CONTEXT.md + 当前相关源码
   → 不要读所有文件，只读与本次修复相关的

3. 说明影响范围
   → 哪些文件会改
   → 哪些用户路径可能受影响
   → 有没有边界风险

4. 执行修改
   → 只改必要文件
   → 不顺手扩大范围

5. 最小验证
   → 跑与本次修复相关的验证
   → 不需要跑全量测试，但要确认修复路径通了

6. 写 session-log + 提交
   → 写一条到 docs/current/session-log.md
   → 原子提交到 main，commit message 写业务意图
```

## session-log 格式

```markdown
## YYYY-MM-DD
- 修复：[一句话描述做了什么]
- 修改文件：[文件路径列表]
- 验证：[做了什么验证，结果如何]
- 风险：[无 / 有（描述潜在风险）]
```

## 禁止事项

- 不得在 Quick Fix 过程中扩大修改范围
- 不得跳过"说明影响范围"这一步
- 不得用 Quick Fix 处理涉及计费、权限、生产数据的修改
- 不得跳过 session-log 记录
